# 伪FL的定义

伪FL是在NF的基础上，针对部分特殊雷型采用FL方式解决。一盘高级一般用到的雷数低于20，[部分特别不擅长标雷的选手](http://saolei.wang/Player/Index.asp?Id=3869)可能只能假假地标一个雷。

# 伪FL的意义

伪FL的目标是在保持NF速度与优势的基础上，在特殊局部采用FL手法爆发式提速，去除破空费操不稳定的问题，提升个人成绩上限。

# 伪FL的基本理论

比较FL与NF的优劣势：

## NF的优势：
- 无需考虑左右键切换，减轻思考负担
- 无需考虑标雷位置、双击位置，减轻思考负担
- 某些方向的信息可以更快暴露，有利于后续决策

## FL的优势：
- 破空费操稳定，各种位置都只需要一次1.5click操作
- 数字1密集区或已有1雷的2周围可以实现标1开N，结合1.5click操作以后只要N>=2就一定获益，效率高
- 标过的雷可以在多个方向多次利用，效率极高
- 标过的雷可以作为判雷基点

## FL的劣势：
- 需要考虑左右键切换，思考负担大
- 需要考虑标雷位置、双击位置，思考负担大
- 所有方向的信息必须等到双击结束才能同时暴露，影响后续决策

## NF的劣势：
- 破空费操不稳定，绝对破空位置只需要1次点击，易破空位虽可能只需要1-2次点击但若不是空就亏损，其余不易破空位置如破空往往需要6-7次点击
- 数字1密集区或已有1雷的2周围需要多次点击才能完全清空，效率低下
- 在一个雷的多个方向都要进行多次点击才能完全清空，效率极为低下
- 不存在已标雷作为判雷基点，视野小、雷型不熟练的玩家容易错过可以解决的局部，影响路线的合理性

## 同时使用NF技术与FL技术的劣势：
- 在NF有利区域同样需要FL的思维负担
- 操作容易变形
- 需要额外增加切换模式或者长按的操作（手机），前者要等按下切换键才能进行下一步操作；后者要在足以与NF短按操作区分的时长（下限，尽可能长）以及长按时长（上限，尽可能短）中进行平衡（阈值一般设置为100毫秒），实际使用长按时，存在长按时间未达阈值，仍将被识别为短按打开操作，从而导致本局失败的风险，因此长按时长往往会比100毫秒更长。无论是点击切换模式再标双（标双结束后还得再次切换回来），还是选择一次长按标雷，所需的时长足以完成连续两次点击，伪FL可以得利的形状变得更少。

综上所述，**伪FL需要尽可能的结合NF与FL的优势，规避以上劣势**。借助**双/多线操作**，以及**基本类型的组合**，可以达到目的。伪FL的本质，仍然是NF，伪FL=FL加成过的NF。

## 隔空标雷 vs 双线

<div align="center"> <img src="https://github.com/putianyi889/Minesweeper-makes-me-happy/blob/main/wiki/images/Key-of-Fake-FL/NF-plus-Double-Thread-FL.png" width = 40% height = 40% /> <img width=10/> <img src="https://github.com/putianyi889/Minesweeper-makes-me-happy/blob/main/wiki/images/Key-of-Fake-FL/NF-plus-Double-Thread-FL-result.gif" width = 40% height = 40% /> </div>

鞠泽恩式双线伪FL操作：从左到右NF三连击绿色三格，随后顺手在c-d接一个标双打开efg三格，然后由于b是1补一次双击，f再补一个双击，再回到h补双击。后续仍然由NF完成。通过NF接FL双线，有效平衡了视野与操作，完美利用了两侧局部对新信息的反应时间，没有废操作。

周丹式伪FL操作：先左键点击a，然后标双c-d直接连f，后续再连双击bh。a-d的连续操作名为**隔空标雷**，虽然节省b位1次点击，但也使得**b位的信息晚一步才暴露**（手机上c位标雷可以换算成2次点击时间，b位信息要晚两步才暴露），后续的双击不得不依靠试探双击f位来抹平b位判断时间，或者卡顿一下判出b才能继续进行，失去了双线的妙味。这可能也是周丹未能开发出鞠泽恩式双线的原因之一。

<div align="center"> <img src="https://github.com/putianyi889/Minesweeper-makes-me-happy/blob/main/wiki/images/Key-of-Fake-FL/Jump-Flag-Best.png" width = 40% height = 40% /> </div>

隔空标雷并非除了好看一无是处，没有双线潜力的时候仍然是最有效的：当雷处于内侧，黄色指针按下位置无法提供更多信息的情况下，可以先点击靠雷墙的3（指针上方）之后选择标双红色雷以及绿底的1，形成标1开4的效果。

总结：**隔空标雷技术效率、稳定性均低于双线，不推荐在开阔区域采用，仅适用于封闭区域**。

## 绕圈双击 vs 双线

<div align="center"> <img src="https://github.com/putianyi889/Minesweeper-makes-me-happy/blob/main/wiki/images/Key-of-Fake-FL/Flag-1-chord-N.png" width = 40% height = 40% /> <img width=10/> <img src="https://github.com/putianyi889/Minesweeper-makes-me-happy/blob/main/wiki/images/Key-of-Fake-FL/Flag-1-chord-N-result.gif" width = 40% height = 40% /> </div>

[田溢昉](https://saolei.wang/Player/Index.asp?Id=24)式暴力FL操作：首先标雷a，然后接双击b打开绿色3格，再双击c-d打开浅绿色6格，最后双击e-f打开黄色5格。此过程称为**绕圈双击**，被双击格除了b已知是1，其余c-f四格都需要靠运气才能起作用，在它们恰好都是1的情况下收益最高，在电脑上收益比为`打开格子数/操作数=14/5.5=2.55`（标雷与首次双击按1.5次操作计算），在手机上收益比为`打开格子数/操作数=14/7=2`（标雷按2次操作计算）。在本例中，双击b-c-d-f均遇到1，双击e无效（点在双击c产生的op边缘上），实际打开7格+1个op。

绕圈双击是上古暴力FL大神田溢昉的独门秘技，是**试探双击**技巧的极端化，b-c-d可以说是安全的试探双击位，而e-f两个位置都是绕圈双击起手时没有信息的格子。绕圈双击的优点是在1特别密集的局部能产生恐怖的爆发力（田溢昉中级3bv/s纪录5.83），缺点是过于依赖1，在烂图容易卡顿，且手机上盲目双击可能点到雷。

效率流对绕圈双击的一个小改进是：**标雷a双击b**后，由于双击d可以直接打开5个新格子，而双击c只能打开3个新格子，两者重叠覆盖的格子有2个，故**先双击d**可以使得更多的格子先暴露，再在**补双击c（效率等价于NF）**的同时判断前述5格带来的信息，**最后补双击f-e**。效率流的双击位置思路来源于对IOE理论最高值的探索，结论是双击雷横竖左右四个方向邻接格效率最高。

<div align="center"> <img src="https://github.com/putianyi889/Minesweeper-makes-me-happy/blob/main/wiki/images/Key-of-Fake-FL/Flag-1-chord-N-to-Double-Thread-FL.png" width = 40% height = 40% /> </div>

我们看看鞠泽恩式双线伪FL在此局部的扫法：NF二连击a-b绿色格，随后顺手在c-d接一个标双打开浅绿色格（此时判出绿色格），然后双击b（此时判出e与e上面一格），再对e补一个双击（此时判出b右侧是否是1），最后对b右侧格补双击（如b右侧是1，不是则不补）。仅讨论确定的部分，省略最后一次双击，与绕圈双击同样的操作数，不仅规避了1的不确定性，同时避免了卡顿和踩雷，代价仅仅是少覆盖了2格，而这可以根据后续情况用1次双击即可覆盖。

总结：**绕圈双击技术稳定性远低于双线，在理想状态下效率也没有明显提升，不推荐采用；试探双击的稳定性低于双线，仅推荐局部收尾（附近没有其他可解局部）时采用**。

## 试探双击 vs 双线

试探双击是李明炫的必杀技，主要作用是：当你完成一个标双还不知道下一步点哪里的时候，沿着上一次双击的惯性再盲目地双击一下，为下一步判断争取时间的同时争取多开几个格子。第1次双击在电脑上由1.5click技巧完成，耗时按0.5次点击计算，此前的标雷耗时1次点击；第1次双击在手机上由1次点击完成，耗时按1次点击计算，此前的标雷耗时2次点击。第2次点击则主要分为两种情形：①在首次双击格与雷横竖相邻的情况下，第2次点击方向与第1次相对于雷垂直，如第2次点击不成功则在其他操作时判断与雷横竖相邻的格能否补双击，②在首次双击格与雷斜向相邻的情况下，在其他操作时优先判断对与雷横竖相邻的格是否需要补双击。

<div align="center"> <img src="https://github.com/putianyi889/Minesweeper-makes-me-happy/blob/main/wiki/images/Key-of-Fake-FL/Prechord.png" width = 40% height = 40% /> <img width=10/> <img src="https://github.com/putianyi889/Minesweeper-makes-me-happy/blob/main/wiki/images/Key-of-Fake-FL/Prechord-result.gif" width = 40% height = 40% /> </div>

这是试探双击的一种极端用法，待解决局部被四面包围，其中绿色格安全，黄色格c-d无法判定是否是雷。由于高级雷密度为20.625%，c为雷的概率约为20.6%，c-d均没有雷的概率就是(1-20.625%)^2=63%。手机上，a-b都是安全的双击位，c有踩雷的风险；电脑上，a-b-c都是安全的双击位。

手机上在a-b-c连续双击过程里安全的概率约是79.4%（可能踩雷的位置只有c，而电脑上无此顾虑）。在概率约63%的理想状态下（c-d都不是雷），手机效率（打开格子数/操作数）为`8/5=1.6`，电脑效率为`8/3.5=2.29`；在概率约16.4%的差一点的情况下（c不是雷，d是雷），手机效率为`7/7=1`，电脑效率为`7/5.5=1.27`，也不亏；在概率为20.6%的最差情况下（c是雷），手机gg，电脑效率为`6/6.5×20.6%+7/7.5×79.4%=0.93`。加权平均的手机效率（生存情况下）为`(1.6×63%+1×16.4%+0×20.6%)/79.4%=1.17/79.4%=1.47`，电脑效率为`2.29×63%+1.27×16.4%+0.93×20.6%=1.84`。

如果操作保守一点，仅双击a-b，概率约79.4%的理想状态下手机效率为`5/4=1.25`，电脑效率为`5/2.5=2`；而概率约20.6%的c为雷状态下，c右侧一格可以通过双击和下面一格同时打开，按半次操作计算，c左侧一格按nf计算，手机效率为`4/5.5=0.73`，电脑效率为`4/4=1`。加权平均的手机效率为`1.25×79.4%+0.73×20.6%=1.14`，电脑效率为`2×79.4%+1×20.6%=1.79`。因此在封闭局部下，无论手机或者电脑，试探双击两次都是有利的。可以考虑的后续操作是单击d下面1格，同时判断c是否可以双击，再补双击c或者改用nf，这样整体概率计算如下：79.4%可能连双击成功，其中又有79.4%概率可以补双击c，20.6%概率nf两格，手机效率为`8/5×79.4%+7/6×20.6%=1.51`，电脑效率为`8/3.5×79.4%+7/4.5×20.6%=2.14`；20.6%连双击失败，右侧两绿格可以通过一个补双击完成，其余格nf完成，手机效率为`7/8×79.4%+6/7×20.6%=0.87`，电脑效率为`7/6.5×79.4%+6/5.5×20.6%=1.08`。加权平均的手机效率为`1.51×79.4%+0.87×20.6%=1.38`，电脑效率为`2.14×79.4%+1.08×20.6%=1.92`。**本方案是局部最优解**。

再考虑nf上方两格，再点开d下面1格，然后标雷连双击两次的方案，其中在第一次连双击时即可根据d下面1格提供的信息判出是否需要双击第二次还是改用nf。为了方便比较不计入d下面1格的效率。在概率约63%的理想状态下（c-d都不是雷），手机效率为`8/6=1.33`，电脑效率为`8/4.5=1.78`；在概率约16.4%的差一点的情况下（c不是雷，d是雷），手机效率为`7/7=1`，电脑效率为`7/5.5=1.27`；在概率为20.6%的最差情况下（c是雷但可以双击解决右侧两绿色格），手机效率为`7/8×79.4%+6/7×20.5%=0.87`，电脑效率为`7/6.5×79.4%+6/5.5×20.5%=1.08`。加权平均的手机效率为`1.33×63%+1×16.4%+0.87×20.6%=1.18`，电脑效率为`1.78×63%+1.27×16.4%+1.08×20.6%=1.55`。**本方案效率明显不如双击a-b再补双击**。

再考虑左侧标雷二连双击+右侧两绿格nf的方案。在概率约63%的理想状态下（c-d都不是雷），手机效率为`8/6=1.33`，电脑效率为`8/4.5=1.78`；在概率16.4%的差一点的情况下（c不是雷，d是雷，还有标1开2的机会），手机效率为`7/7=1`，电脑效率为`7/5.5=1.27`；在概率16.4%的次差情况下（c是雷，d不是雷，电脑上还可以用1.5click标双c），手机效率为`7/8=0.875`，电脑效率为`7/6=1.17`；在概率4.2%的最差情况下（c-d都是雷），手机效率为`6/7=0.86`，电脑效率为`6/5.5=1.09`。加权平均的手机效率为`1.33×63%+1×16.4%+0.875×16.4%+0.86×4.2%=1.18`，电脑效率为`1.78×63%+1.27×16.4%+1.17×16.4%+1.09×4.2%=1.57`。**本方案效率明显不如双击a-b再补双击**。电脑上如改用1.5click技巧处理右侧两绿格，效率为`8/4×63%+7/5×16.4%+7/5.5×16.4%+6/5×4.2%=1.75`，虽然略有提升，但仍然不如双击a-b再补双击的方案。

如果两边标雷又会如何呢？先左侧标双一次，然后右侧标双的同时判出c-d，然后再选择双击位置或单击b格。在概率约63%的情况下（c-d都不是雷），手机效率为`8/7=1.14`，电脑效率为`8/4=2`；在概率16.4%的差一点的情况下（c不是雷，d是雷，还有标1开2的机会），手机效率为`7/7=1`，电脑效率为`7/4=1.75`；在概率16.4%的次差情况下（c是雷，d不是雷，电脑上还可以用1.5click标双c），手机效率为`7/8=0.875`，电脑效率为`7/4.5=1.56`；在概率4.2%的最差情况下（c-d都是雷），手机效率为`6/7=0.86`，电脑效率为`6/4=1.33`。加权平均的手机效率为`1.14×63%+1×16.4%+0.875×16.4%+0.86×4.2%=1.06`，电脑效率为`2×63%+1.75×16.4%+1.56×16.4%+1.33×4.2%=1.86`。本方案由于手机在右侧标1开2效率低下，严重拉低了整体效率，而**电脑上的效率比三连双击还要高，是次优解**。

最后考虑从右侧三连双击到c的方案，在第二次双击失败的情况下仍然可以双击a。在概率约63%的理想状态下（c-d都不是雷），手机效率为`8/5=1.6`，电脑效率为`8/3.5=2.29`；在概率16.4%的差一点的情况下（c不是雷，d是雷），手机效率为`7/8=0.875`，电脑效率为`7/7.5=0.93`；在概率16.4%的次差情况下（c是雷，d不是雷），手机gg，电脑效率为`7/7.5=0.93`；在概率4.2%的最差情况下（c-d都是雷），手机gg，电脑效率为`6/6.5=0.92`。加权平均的手机效率（生存情况下）为`(1.6×63%+0.875×16.4%+0×20.6%)/79.4%=1.15/79.4%=1.45`，电脑效率为`2.29×63%+0.93×16.4%×2+0.92×4.2%=1.79`。**本方案效率不如a-b-c三连双击**，这是由于原方案**将不确定性分担到两次双击中**，而本方案是一锤子买卖。

小结：**在封闭局部且首次双击格与雷斜向相邻时，无论手机或者电脑，试探双击两次都是有利的**。手机试探第三次时看似用筛图的方式略微提高了效率（1.38->1.47），但实际效率是下降的（1.38->1.17），而电脑上效率也是略有下降（1.84->1.79），因此**手机上不建议连双到第三次，电脑上重视操作连续性的选手可以采用三连双击**。本局部的**最优手段是标连双2次，单击d下面1格，同时判断c是否可以双击，再补双击c或者改用nf**。

半封闭局部：由于李明炫已经点出一个2，确定了浅绿格无雷，本局部的风险性进一步降低，仅剩两个黄格不确定，如果事先标双了下方的标1开4，两黄格也变为确定格，就变成上述的封闭局部。因此本局部也采用了试探双击的极端用法，标雷后连双击a-b-c。由于都是在双击b-c时会各碰到一个未定格，故概率分析与封闭局部相似：

连双击a-b-c，在概率约63%的理想状态下，手机效率为`7/5=1.4`，电脑效率为`7/3.5=2`；在概率约16.4%的差一点的情况下，手机效率为`4/5=0.8`，电脑效率为`4/3.5=1.14`；在概率为20.6%的最差情况下，手机效率为`2/5=0.4`，电脑效率为`2/3.5=0.57`。加权平均的手机效率为`1.4×63%+0.8×16.4%+0.4×20.6%=1.1`，电脑效率为`2×63%+1.14×16.4%+0.57×20.6%=1.56`。

双击a-b后面再补双击c或nf，在概率约63%的理想状态下，手机效率为`7/5=1.4`，电脑效率为`7/3.5=2`；在概率约16.4%的差一点的情况下，手机效率为`6/7=0.86`，电脑效率（对2补1.5click）为`6/5=1.2`；在概率约16.4%的次差情况下，手机效率为`6/7=0.86`，电脑效率（对2补1.5click）为`6/5=1.2`；在概率为4.2%的最差情况下，手机效率为`5/7=0.71`，电脑效率（对2补1.5click）为`5/4=1.25`。加权平均的手机效率为`1.4×63%+0.86×16.4%+0.86×16.4%+0.71×4.2%=1.19`，电脑效率为`2×63%+1.2×16.4%+1.2×16.4%+1.25×4.2%=1.71`。**这是本局部的最优解**。

仅双击a-b不补双击，概率约79.4%的理想状态下，手机效率为`4/4=1`，电脑效率为`4/2.5=1.6`；而概率约20.6%的c下方为雷的状态下，手机效率为`2/4=0.5`，电脑效率为`2/2.5=0.8`。加权平均的手机效率为`1×79.4%+0.5×20.6%=0.9`，电脑效率为`1.6×79.4%+0.8×20.6%=1.44`。由于不补双击，外侧三格看作是nf完成的，效率为1，以上效率在与1作加权平均后，总体效率将往1方向靠近，因此效率将低于连双击a-b-c。

小结：**在半封闭局部只能双击出4格且首次双击格与雷斜向相邻时，无论手机或者电脑，试探双击两次后面再补双击或nf都是比试探三次有利的，但试探双击两次后如不补双击则是亏损的**。

我们再考虑这个2没点出的情况，这样浅绿色与黄色4格都无法确定。手机上，a-b都是安全的双击位，c有20.6%踩雷的风险；电脑上，a-b-c都是安全的双击位。双击b位成功的概率是63%，而双击b位成功后双击c位成功的概率也是63%，故a-b-c连双击都成功的概率是`63%×63%=39.7%`。

连双击a-b-c，在概率约39.7%的理想状态下，手机效率为`8/5=1.6`，电脑效率为`8/3.5=2.29`；在概率约63%×37%=23.3%的差一点的情况下，手机效率为`7/7×88.5%+6/6×11.5%=1`，电脑效率为`7/5.5×88.5%+6/4.5×11.5%=1.28`；在概率约16.4%的次差情况下，手机除了绿色二格还能点开c格，手机效率为`6/8×63%+5/7×16.4%×2+4/6×4.2%=0.73`，电脑打开绿色二格，电脑效率为`6/7.5×63%+5/6.5×16.4%×2+4/5.5×4.2%=0.79`；在概率为20.6%的最差情况下，手机在c格踩雷gg，电脑效率为`7/8.5×50%+6/7.5×10.1%+5/6.5×39%+4/5.5×0.9%=0.80`。加权平均的手机效率（生存情况下）为`(1.6×39.7%+1×23.3%+0.73×16.4%+0×20.6%)/79.4%=0.988/79.4%=1.24`，电脑效率为`2.29×39.7%+1.28×23.3%+0.79×16.4%+0.80×20.6%=1.50`。

双击a-b后面再补c或nf，在概率约39.7%的理想状态下，手机效率为`8/5=1.6`，电脑效率为`8/3.5=2.29`；在概率约23.3%的差一点的情况下，手机效率为`7/7×88.5%+6/6×11.5%=1`，电脑效率为`7/5.5×88.5%+6/4.5×11.5%=1.28`（`(79.375%+79.375%)/(79.375%+79.375%+20.625%)=88.5%`）；在概率约32.8%的次差情况下，手机效率为`7/8×63%+6/7×16.4%×2+5/6×4.2%=0.87`，电脑效率为`7/6.5×63%+6/5.5×16.4%×2+5/4.5×4.2%=1.08`；在概率为4.2%的最差情况下，手机效率为`5/7×63%+4/6×16.4%×2+3/5×4.2%=0.69`，电脑效率为`5/5.5×63%+4/4.5×16.4%×2+3/3.5×4.2%=0.9`。加权平均的手机效率为`1.6×39.7%+1×23.3%+0.87×32.8%+0.69×20.6%=1.30`，电脑效率为`2.29×39.7%+1.28×23.3%+1.08×32.8%+0.9×20.6%=1.75`。**这是本局部的最优解**。

仅双击a-b不补双击，概率约63%的理想状态下，手机效率为`5/4=1.25`，电脑效率为`5/2.5=2`；而概率约37%的双击失败状态下手机效率为`2/4=0.5`，电脑效率为`2/2.5=0.8`。加权平均的手机效率为`1.25×63%+0.5×37%=0.97`，电脑效率为`2×63%+0.8×37%=1.56`。由于不补双击，外侧三格看作是nf完成的，效率为1，以上效率在与1作加权平均后，总体效率将往1方向靠近，因此效率将低于连双击a-b-c。

若故意把2点出再标雷连双击a-b，概率约79.4%的理想状态下，手机效率为`5/5=1`，电脑效率为`5/3.5=1.43`；而概率约20.6%的c下方为雷的状态下，手机效率为`3/5=0.6`，电脑效率为`3/3.5=0.86`。加权平均的手机效率为`1×79.4%+0.6×20.6%=0.92`，电脑效率为`1.43×79.4%+0.86×20.6%=1.31`。**均不如直接标雷连双击a-b的效率**。

若故意把2点出再标雷连双击a-b-c，在概率约63%的理想状态下，手机效率为`8/6=1.33`，电脑效率为`8/4.5=1.78`；在概率约16.4%的差一点的情况下，手机效率为`5/6=0.83`，电脑效率为`5/4.5=1.11`；在概率为20.6%的最差情况下，手机效率为`3/6=0.5`，电脑效率为`3/4.5=0.66`。加权平均的手机效率为`1.33×63%+0.83×16.4%+0.5×20.6%=1.08`，电脑效率为`1.78×63%+1.11×16.4%+0.66×20.6%=1.44`。**均不如直接标雷连双击a-b的效率**。

小结：**在半封闭局部且首次双击格与雷斜向相邻时，无论手机或者电脑，试探双击三次都是有利的**，但手机上有20.6%概率gg。在首次双击格与雷斜向相邻且只能双击出5格时，手机上试探双击两次是亏损的，但电脑上可以这么选择。**没有必要为了额外的信息量多点一格**。

开放式有已标雷局部：这是试探双击的一种正常用法，利用已标雷连续双击a-b，同时判断下方最优路线。下方NF过程（1-2）同时判出上方c位可以补双击，在下方解决到与上方平齐时（3-8）再补9位，然后再破空10-13。**下方的路线是根据深度优先原则制定的，每一步都要求获取右侧最大信息量**，通过每个格获取到的新信息已用绿线标注在图中。

由于本局部是开放式的，c格一列的浅绿两格都有可能是雷。由于高级雷密度为20.625%，c为雷的概率约为20.6%，浅绿格均没有雷的概率就是63%。黄格由于是在其他过程中判出可以双击的，属于净收益，效率是3，可以双击的概率是(1-20.625%)^3=50%。

双击a-b-c，概率约31.5%的理想情况（双击三次均成功）下效率为`7/3=2.33`；概率约31.5%的次优情况（双击两次成功）下效率为`4/3=1.33`；概率约16.4%的次差情况（双击一次成功且最后一次未踩雷）下手机效率为`3/3=1`，电脑效率为`2/3=0.67`；概率约20.6%的最差情况（双击一次成功且最后一次双击在雷上）下手机gg，电脑效率为`2/3=0.67`。加权平均的手机效率（生存情况下）为`(2.33×31.5%+1.33×31.5%+1×16.4%+0×20.6%)/79.4%=1.32/79.4%=1.66`，电脑效率为`2.33×31.5%+1.33×31.5%+0.67×16.4%+0.67×20.6%=1.40`。

仅双击a，效率为`2/1=2`。另外还有63%的净收益可能，效率也是2，因此总体效率仍然是2。

仅双击a-b不补双击c，概率约63%的理想状态下效率为`7/3×50%+4/2×50%=2.17`；而概率约37%的其他状态下效率为`3/3=1`。加权平均的效率为`2.17×63%+1×37%=1.74`。

小结：**在开放局部存在已标雷且首次双击格与雷斜向相邻时，无论手机或者电脑，双击一次，或者试探双击两次再补双击都是有利的**；连双击三次在手机和电脑上仍然有利，但手机上有20.6%概率gg，电脑上效率下降严重，稳定性不如前者，仅推荐在缺少可解局部的情况下采用。
